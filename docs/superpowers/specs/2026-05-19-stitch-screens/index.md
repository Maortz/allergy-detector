# Stitch Screens вАФ Spec Tracker

**Single source of truth** for every screen's status across five dimensions:
Stitch design ¬Ј Spec ¬Ј Implementation ¬Ј Verification-vs-Spec ¬Ј Verification-vs-Art.
Status is tracked **only here** вАФ no status table is duplicated in sibling files.

**Column legend:**
- **Stitch** вЬУ = Stitch art generated (Screen ID exists) ¬Ј вЬЧ = not drawn
- **Spec** вЬУ = own `.md` spec file here ¬Ј вЧР = no standalone file, specified inside a parent screen's `¬І` section
- **Code** вЬУ = Dart impl shipped to `master` ¬Ј вЬЧ = not implemented ¬Ј вЧС = art drawn, code pending
- **V-Spec** (impl vs spec doc) вЬУ = aligned ¬Ј вЪ† = diverged (see spec ¬І7) ¬Ј вђЬ = impl exists, not checked ¬Ј вАФ = N/A
- **V-Art** (impl vs Stitch rendered art, via `get_screen <id>`) вЬУ = matches art ¬Ј вЪ† = diverged ¬Ј вђЬ = impl exists, not checked ¬Ј вАФ = N/A

> **Why two verification columns:** specs are a *prescriptive transcription* of the
> Stitch art, and the design decisions (`_design-decisions.md` DD-1..DD-17)
> deliberately override the art in places. So an impl can match the spec (V-Spec вЬУ)
> yet still differ from the raw art (V-Art вЪ†), or vice-versa. Track both.

# Stitch Screens вҖ” Spec Tracker

**Single source of truth** for every screen's status across five dimensions:
Stitch design В· Spec В· Implementation В· Verification-vs-Spec В· Verification-vs-Art.
Status is tracked **only here** вҖ” no status table is duplicated in sibling files.

**Column legend:**
- **Stitch** вң“ = Stitch art generated (Screen ID exists) В· вң— = not drawn
- **Spec** вң“ = own `.md` spec file here В· в—җ = no standalone file, specified inside a parent screen's `В§` section
- **Code** вң“ = Dart impl shipped to `master` В· вң— = not implemented В· в—‘ = art drawn, code pending
- **V-Spec** (impl vs spec doc) вң“ = aligned В· вҡ  = diverged (see spec В§7) В· в¬ң = impl exists, not checked В· вҖ” = N/A
- **V-Art** (impl vs Stitch rendered art, via `get_screen <id>`) вң“ = matches art В· вҡ  = diverged В· в¬ң = impl exists, not checked В· вҖ” = N/A

> **Why two verification columns:** specs are a *prescriptive transcription* of the
> Stitch art, and the design decisions (`_design-decisions.md` DD-1..DD-17)
> deliberately override the art in places. So an impl can match the spec (V-Spec вң“)
> yet still differ from the raw art (V-Art вҡ ), or vice-versa. Track both.
